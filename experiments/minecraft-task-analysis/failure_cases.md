# Failure Cases

## Planning Failures

- Missing prerequisite resources
- Incorrect tool requirements
- Overly broad subtask names
- No recovery step after inventory loss

## Perception Failures

- Misidentifying blocks
- Losing sight of crafted items
- Failing to detect threats

## Control Failures

- Poor navigation in caves
- Falling damage or lava damage
- Inefficient mining patterns

## Evaluation Notes

Each failure case should record the task, context, expected behavior, observed behavior, and likely root cause.
